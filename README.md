# Evaluación Adaptativa de Python

## Descripción

Este recurso es una aplicación web educativa desarrollada con HTML, CSS y JavaScript que realiza una evaluación adaptativa de Python. El alumnado responde primero a un bloque de preguntas diagnósticas y, en función de sus resultados, accede a un itinerario de preguntas de nivel Básico, Intermedio o Avanzado. Al finalizar, recibe una nota, un informe detallado y recomendaciones personalizadas para mejorar su aprendizaje.

---

## Objetivo didáctico

El objetivo de esta actividad es evaluar el nivel de competencia del alumnado en programación con Python, proporcionando una experiencia adaptativa que permita reforzar contenidos básicos, intermedios o avanzados según los conocimientos demostrados durante la prueba. Además de calificar, la herramienta fomenta la reflexión sobre los errores mediante explicaciones y recomendaciones personalizadas.

## Estructura de la actividad

### Fase 1: Diagnóstico inicial

El alumnado responde a 5 preguntas orientadas a detectar su nivel de conocimientos.

Posibles resultados:

- 🟢 Básico
- 🟠 Intermedio
- 🔴 Avanzado

---

### Fase 2: Evaluación adaptada

Según el nivel detectado se presentan:

- 5 preguntas adaptadas al nivel obtenido.

---

### Fase 3: Consolidación

Posteriormente se presentan:

- 5 preguntas adicionales diferentes al bloque anterior.
- Ajustadas al mismo nivel del alumnado.

---

### Informe final

Al finalizar se muestra:

- Nivel detectado.
- Nota final sobre 10.
- Historial completo de respuestas.
- Respuestas correctas.
- Explicaciones teóricas.
- Recomendaciones de estudio.
- Botón para repetir la evaluación.

---

## Tipos de preguntas incluidos

El recurso combina diferentes formatos:

- Preguntas de opción múltiple.
- Completar código.
- Corregir errores.
- Escritura de código.
- Interpretación de instrucciones básicas de Python.

---

## Instrucciones de uso en clase

1. Explicar al alumnado que las primeras cinco preguntas sirven únicamente para determinar su nivel inicial.
2. Permitir que cada estudiante complete la evaluación de forma individual.
3. Solicitar que revisen atentamente el informe final y las recomendaciones obtenidas.
4. Utilizar los resultados para detectar dificultades comunes del grupo.
5. Proponer actividades de refuerzo diferenciadas según el nivel detectado.

---

## Cómo probarlo en local

### Opción 1: Abrir directamente

1. Descargar o copiar todos los archivos del proyecto.
2. Mantener la estructura de carpetas indicada.
3. Abrir el archivo `index.html` con cualquier navegador moderno.

### Opción 2: Servidor local

Desde la carpeta del proyecto:

```bash
python -m http.server 8000
```
